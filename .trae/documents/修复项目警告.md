## 修复项目警告计划

### 1. 修复 Next.js 构建警告
- **问题**：Next.js 构建过程中检测到多个 lockfiles，推断的工作区根目录可能不正确
- **解决方案**：在 `next.config.ts` 中明确设置 `turbopack.root` 来指定工作区根目录
- **文件修改**：`next.config.ts`

### 2. 优化 vote-admin.html 文件
- **问题**：
  - Supabase API 密钥硬编码在客户端代码中（安全问题）
  - 使用了过时的库版本（如 SheetJS xlsx-0.19.3）
  - 代码结构可以进一步优化
- **解决方案**：
  - 考虑将敏感信息移到更安全的位置
  - 更新库版本到最新稳定版
  - 优化代码结构和语法
- **文件修改**：`public/vote-admin.html`

### 3. 检查并优化其他文件
- **问题**：确保所有文件符合最佳实践
- **解决方案**：检查 `app/layout.tsx` 和 `app/page.tsx` 等文件，添加必要的类型注释和优化
- **文件修改**：`app/layout.tsx`、`app/page.tsx`

### 4. 运行验证命令
- **问题**：确保所有警告都已修复
- **解决方案**：运行 `npm run build`、`npx tsc --noEmit` 和 `npx eslint .` 来验证修复效果

### 执行顺序
1. 首先修复 `next.config.ts` 中的 Next.js 构建警告
2. 然后优化 `vote-admin.html` 文件
3. 接着检查并优化其他文件
4. 最后运行验证命令确保所有警告都已修复
